# OptimisedEats 🥗

**Eat better for less — personalised to your household.**

Live at [optimisedeats.com](https://optimisedeats.com)

A React app that builds personalised meal plans for Australian and New Zealand families, optimised for both nutrition and budget. It knows each household member's age, sex, and diet type, calculates their daily nutrient requirements, and flags gaps and supplement needs.

---

## Project Structure

```
budget-nutrition-app-LATEST.jsx   ← MASTER SOURCE — edit this, never App.jsx
build_v2.py                        ← Build processor (LATEST → App.jsx)
guide-pages/                       ← Static guide hub HTML pages + generator
budget-nutrition-au-project/       ← Vite project
  src/App.jsx                      ← GENERATED by build_v2.py (not tracked)
  package.json
  vite.config.js
```

## Development Workflow

### 1. Make changes
Edit `budget-nutrition-app-LATEST.jsx` only. Never edit `src/App.jsx` directly — it gets overwritten on every build.

### 2. Build
```bash
python build_v2.py
```
This processes LATEST.jsx → App.jsx and runs `vite build`.

### 3. Commit
```bash
git add budget-nutrition-app-LATEST.jsx
git commit -m "feat: describe what you changed"
git push origin dev
```

### 4. Deploy to production
```bash
# Patch index.html with new JS filename, create deploy zip, SCP to server
python -c "
import zipfile, re
NEW_JS = 'index-XXXXXXXX.js'   # replace with actual filename from dist/assets/
with zipfile.ZipFile('budget-nutrition-au-dist-full.zip','r') as z:
    hero = z.read('index.html').decode('utf-8')
    old_js = re.search(r'assets/(index-[^\"]+\.js)', hero).group(1)
    hero_patched = hero.replace(old_js, NEW_JS)
with zipfile.ZipFile('deploy.zip','w', compression=zipfile.ZIP_DEFLATED) as z:
    z.writestr('index.html', hero_patched.encode('utf-8'))
    z.write(f'budget-nutrition-au-project/dist/assets/{NEW_JS}', f'assets/{NEW_JS}')
"

scp -i "ssh-key-2026-04-21 (1).key" deploy.zip ubuntu@168.138.23.164:/tmp/deploy.zip
ssh -i "ssh-key-2026-04-21 (1).key" ubuntu@168.138.23.164 \
  "sudo unzip -o /tmp/deploy.zip -d /var/www/optimisedeats/ && \
   sudo chown www-data:www-data /var/www/optimisedeats/index.html /var/www/optimisedeats/assets/*.js && \
   sudo chmod 644 /var/www/optimisedeats/index.html /var/www/optimisedeats/assets/*.js"
```

### 5. Merge to main after deploy
```bash
git checkout main
git merge dev
git push origin main
git checkout dev
```

## Server Details

- **Host:** Oracle Cloud Ubuntu — `ubuntu@168.138.23.164`
- **Web root:** `/var/www/optimisedeats/`
- **Nginx config:** `/etc/nginx/sites-enabled/optimisedeats`
- **⚠️ After every deploy:** run `chown www-data:www-data` + `chmod 644` on new files — `sudo unzip` creates root-owned files that nginx can't read

---

## Tech Stack

- **Frontend:** React 18 + Vite
- **Styling:** Inline styles (dark theme)
- **State:** useState / useLocalStorage hooks
- **Data:** All recipes, NRV tables, and pricing data are embedded in the source file
- **Hosting:** Oracle Cloud Ubuntu + nginx (static files only — no backend)
